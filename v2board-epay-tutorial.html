常见问题

❓ URL 应该填什么？要加 /submit.php 吗？
V2Board 的 EPay 驱动会自动在 URL 后拼接 /submit.php，所以填 https://api2.188pay.top 即可，不需要手动加 /submit.php，也不要加末尾斜杠。
❓ 支付成功了，但订单还是待支付状态？
检查「系统配置 → 站点网址」是否填了正确的公网域名或 IP，并且 V2Board 可以被公网访问。本地或内网环境无法收到 188pay 的支付回调。回调地址在支付配置列表的「通知地址」列可以查看。
❓ 支付页面提示「签名错误」？
请检查 PID 和 KEY 是否从 188pay 完整复制，粘贴时不要包含多余的空格或换行符。也可以尝试重新生成密钥后再填一次。
❓ 用户看不到 188pay 支付选项？
确认支付配置列表中 188pay 的启用开关已打开（蓝色状态）。另外检查该支付方式对应的订阅套餐是否有关联的支付配置限制。

错误排查

! 常见错误排查：
  • • URL 格式错误：只填 https://api2.188pay.top，不要加 /submit.php，也不要末尾斜杠。
  • • 站点网址未配置：「系统配置 → 站点网址」必须是可公网访问的真实域名或 IP，否则 188pay 无法回调。
  • • 接口文件选错：请确认下拉选了「EPay」，而不是其他驱动（Stripe、PayPal 等协议不兼容）。
  • • 收款地址未配置：在 188pay 商户中心确认已添加 USDT（TRC20）或 TRX 收款地址，并已启用对应币种。
  • • PID / KEY 带有空格：复制时可能夹带多余空格，粘贴后仔细检查首尾是否干净。
